# Push Strategy — Overview

## Strategic Definition

Push is **Vertical AI for Local Commerce**. Merchants tell Push how many new customers they want; Push's system matches creators, executes campaigns, verifies results, and optimizes.

**Push IS:** transaction system, trust system, creator credit system, local demand orchestration system.
**Push is NOT:** generic influencer marketplace, local discovery app, content community, reward app, loyalty system.

## Core Insight

Creators are not media inventory—they are **unstructured customer acquisition channels**. The opportunity is to standardize, verify, rank, and route creator-driven demand. This is infrastructure, not a marketplace.

## Five Strategic Pillars

1. **Offer OS:** Merchants define growth goals → system converts to structured campaigns (launch item, drive off-peak, fill event, increase first-visits, test menu)
2. **Creator Credit:** Valued by completion rate, reliability, satisfaction, repeat likelihood, dispute history, geography, category fit—NOT just followers
3. **Attribution:** QR-code transaction-level verification; 30-day window
4. **Demand Surface:** Real-time interface showing opportunities, closing times, qualifications, payments, scarcity
5. **Liquidity Layer:** Standby mechanism increases slot fill, engagement, urgency, merchant confidence

## Company-Building Thesis

**End state:** A new market layer standardizing creator-driven local commerce. Operating system positioning—Push is not a platform (replaceable) but an **OS** (irreplaceable due to accumulated intelligence: creator scores, merchant playbooks, matched data). Category: **Vertical AI for Local Commerce**. Benchmark: Shopify for e-commerce = Push for local creator commerce.

## Core Invariant

Push only rewards **verified, repeatable value creation**. Three tests: Can it be verified? Can it be repeated? Does it create real value?

## Prioritization Framework

| Priority | Examples |
|----------|----------|
| **Highest** | Verifiable campaign outcomes, creator scoring, repeat merchant workflow, creator retention via progression, anti-disintermediation, standby mechanics |
| **Medium** | Premium membership, advanced analytics, vertical templates, QoL automation |
| **Low** | Social/community features, broad creator expression, ornamental loyalty |

## Decision Standard

Every major decision must answer YES to at least one:
1. Improves merchant ROI clarity?
2. Improves creator earning power or quality sorting?
3. Deepens platform defensibility?
4. Reduces fraud, friction, or repeat campaign cost?
5. Makes future transactions more likely to stay on-platform?

## Strategic Red Flags

Push is drifting if it behaves like: sample-giveaway app, creator coupon engine, posting board, local discovery map with weak economics, content engagement platform with shallow verification.
