# Agentic Roadmap — Push 2026–2027 (v5.1)

> **v5.1 hardening (2026-04-18):** AI is **Day-1**, not gated. The old "Phase 0 manual → Phase 1 AI after 50+ campaigns" staging is fully retired — v5.0 removed the gate, v5.1 hardens what replaces it. On campaign #1 every claimed visit runs through multi-modal verification, every post runs through **DisclosureBot**, every verified event feeds **ConversionOracle™**, and every creator slate is generated by the matching agent. Humans handle strategy, relationships, and ~20% exception review. This file documents the Day-1 stack, the ConversionOracle training flywheel, DisclosureBot compliance architecture, and the autonomy ladder that follows.

## Why Agentic (v5.1 Framing)

**SaaS Winter.** General AI (Claude, GPT, Gemini) commoditizes software features. "SaaS tool with AI sprinkled on" positioning dies — anyone can clone a feature list in a weekend.

**Push survives because it prices on outcomes and compounds on proprietary data.** The unit of sale is a **verified customer**, not a seat. AI is the execution layer and the data-capture layer. Competitors cloning the Vision + OCR + geo pipeline still need 1,000+ verified events before their matching model trains — and by then Push has head-started its next vertical. They cannot rent walk-in ground truth.

**Position (v5.1 external):** Push is **Vertical AI for Local Commerce** — a Customer Acquisition Engine that AI-operates a creator network against verified walk-in outcomes. The YC RFS #5 framing (AI-Powered Agencies) is how YC classifies the space internally; Push's outward language is "Vertical AI" / "Customer Acquisition Engine," never "agency." This matters because venture partners treat "agency" as a scale-ceiling alarm word, and Push's Software Leverage Ratio target (M12 ≥25 vs ops shops 3–5) is the direct rebuttal.

## Day-1 AI Architecture (Live from Campaign #1)

There is no manual phase. On campaign #1, merchants experience a system where every claimed visit goes through multi-modal verification and every creator slate is generated by the matching agent. Humans handle exceptions (~20%), not the hot path.

### Day-1 Verification Stack (runs on every claimed visit)

| Layer | Model / Method | What It Checks | Signal |
|-------|----------------|-----------------|--------|
| **Vision** | Claude Vision (Sonnet 4.5 multimodal) | Storefront match, receipt legibility, in-store photo authenticity | Visual confidence 0–1 |
| **OCR** | Vision + text extraction | QR-encoded campaign ID matches claim; receipt item/amount match offer | Hard / soft / fail |
| **Geo** | Geofence + device coords | Scan location within merchant radius; movement plausibility | Within-fence bool + velocity check |
| **Fraud** | Account + device velocity, graph signals | Repeat-account clusters, bot-pattern cadence, impossible timelines | Risk 0–1 |
| **Aggregation** | Weighted combination | Auto-approve / auto-reject / human-review buckets | Decision + confidence |

**Thresholds (v5.1 targets, tuned at campaign-1000 data):**
- Auto-approve: combined confidence ≥ 0.85 (Month 6 target ≥88%)
- Human review: 0.50 – 0.85 (creator relations + merchant ops triage; target ≤20% M3, ≤15% M5)
- Auto-reject: < 0.50 (with appeal path)
- False-positive target <2%; false-negative target <5%

**Day-1 mix:** ~80% auto-decide, ~20% human override. The 20% is a deliberate quality floor, not a "we haven't built AI yet" excuse.

### Day-1 DisclosureBot (FTC 16 CFR Part 255 Platform-Level Enforcement)

Every Push creator post passes through DisclosureBot before publish. This is architectural compliance, not creator-self-reporting compliance.

```
Creator drafts post → CreatorGPT pre-screen →
  ├─ Missing #ad / #sponsored / #paidpartnership? → Block + revision prompt
  ├─ Disclosure buried below fold in caption?    → Warn + reposition suggestion
  ├─ Platform-specific (IG / TikTok / YT) rules? → Auto-rewrite to channel convention
  └─ Pass → Approve + timestamp to audit log
```

**Enforcement:** Creator ToS — bypassing DisclosureBot forfeits payout + drops one tier + second offense = ban. Audit cadence: weekly AI re-check of 10% random sample, monthly flagged-post review, quarterly 1% full human audit with exportable trail, annual external counsel audit. **$1M E&O insurance (Hiscox/Chubb) from Month 6, $25K Y1 legal reserve.**

Competitive note: Fohr / Aspire / Grin rely on creator self-reporting. Push is the only creator platform with **architecturally compliant** disclosure, which is a sellable differentiator to enterprise merchant procurement/legal.

### Day-1 Agentic Campaign Engine

Merchant submits a goal ("20 new customers, Williamsburg, Coffee+, within 7 days") and in ~60 seconds the agent returns:
- A creator slate (5–15 matched operators, ranked by tier + score + geo + Coffee+ affinity + historical conversion)
- A draft campaign brief (offer structure, content guidelines, timing window, vertical-matched payout per verified customer)
- A ConversionOracle-scored ROI prediction (expected verified customers, confidence band, spend estimate)

Merchant approves / edits → agent dispatches → verification stack runs on each claim → DisclosureBot gates publish → payout on auto-approve (48h).

From day one. Not after 50 campaigns.

### Why "Day-1 AI" Is Defensible, Not Theater

Day-1 AI works because:
1. Verification models (Claude Vision, OCR, geo) are **general-purpose** — they work with zero Push-specific training data on day one.
2. The **matching agent** starts with heuristic priors (tier + geo + vertical + content style) and improves with every campaign. It is never "dumb" — it just gets smarter.
3. Fraud detection is seeded with **published adversarial patterns** (bot-farm signatures, geo-impossibility heuristics), then refines.
4. DisclosureBot is rule-based v0 (pattern-match required tokens) shipping Week 1–2, AI-assisted v1 by Month 3.
5. Human review is a **quality floor**, not a crutch — it exists even at scale for edge cases and appeals.

What we do **not** claim on day one: "Our matching agent is smarter than a human buyer." What we do claim: "Our matching agent returns a slate in 60 seconds, end-to-end verified, billed per customer, with FTC compliance architecturally enforced." Those claims survive a partner meeting.

## ConversionOracle™ — The Core Moat Asset

Claude Vision + OCR + geo are features; their outputs are commodity. What is not commodity is the **labeled walk-in ground truth** every verified event produces. This is ConversionOracle™, and it is the v5.1 answer to "AI verification isn't a moat."

### Training data schema

```
Features: (creator_profile, content_style, content_timestamp,
           merchant_category, merchant_neighborhood, offer_type,
           weather, time_of_day, day_of_week)
Label:    walk_in_verified = 1 · customer_LTV = $X
```

### Training flywheel

| Phase | Verified events (cumulative) | Model state | Walk-in prediction error |
|-------|------------------------------|-------------|---------------------------|
| **Day 1–60** (Williamsburg Coffee+, Template 0) | **500 → 1,000** | Cold-start: Claude API + rule-based priors | Heuristic baseline |
| **Month 3–6** | **1,000 → 5,000** | **ConversionOracle v1** fine-tune | **±25%** |
| **Month 6–12** (NYC Dense, 5 neighborhoods) | **10,000 → 25,000** | **v2** walk-in + LTV prediction | **±15%** walk-in, ±30% LTV |
| **Month 12–24** (Top-5 Metro, 30 neighborhoods) | **50,000 → 200,000** | **v3** cohort-specific | **±10%** walk-in, 30-second first-campaign ROI prediction for any new merchant |

### Four structural locks

1. **Data exclusivity.** Walk-in ground truth is not available to any general ads platform. Meta and Google cannot see storefront visits. Apple Wallet has only large chains. Push's exclusivity holds until someone else runs 1,000+ verified events per vertical — a 6–12 month minimum window per category.
2. **Cold-start barrier.** A new entrant needs ~1,000 verified events before v1 trains. That requires its own merchant and creator network already operational, which is the chicken-egg Push solved with the Neighborhood Playbook.
3. **Long-tail advantage.** Large-lab general models are data-sparse on nano/micro creators; Push's specialist model is denser precisely where the economics matter (T1–T3 operators).
4. **Compounding cross-neighborhood transfer.** After 10 neighborhoods, cross-neighborhood priors accelerate Template N launch — Nolita Coffee+ benefits from Williamsburg Coffee+ signal, then NYC-wide Coffee+ benefits from Top-5 Metro Coffee+ signal.

### A/B commitment (diligence-grade)

Month 6 commitment on record: 100-campaign A/B between **ConversionOracle v1 matching** and **zero-shot Claude API matching**. Measured lift on verified-walk-in conversion must be ≥15%. If lift <15%, narrative pivots honestly to ops leverage + supply density + DisclosureBot as primary defensibility — we do not force the data-moat story past what the data supports.

## Autonomy Ladder (Evolves With Data)

Autonomy increases with campaign volume. What does **not** change: verification is Day-1, DisclosureBot is Day-1, billing is per-customer, ConversionOracle captures every event.

### Level 1 — Foundation (Campaigns 1–50)

- **Verification:** Day-1 multi-modal stack live; ~80% auto, 20% human review.
- **Disclosure:** DisclosureBot v0 (rule-based token check) live Week 1–2.
- **Matching:** Agent uses heuristic priors (tier + geo + Coffee+ fit). Humans spot-check every slate before dispatch.
- **Campaign design:** Agent drafts brief; Jiaming / Prum edits before merchant sees it.
- **Ops cost per campaign:** ~3–5 human hours.
- **Software Leverage Ratio:** 8 (M3 target)

**Exit signal:** 50 campaigns, ≥2 merchants at 3+ campaigns, fraud patterns mapped, auto-verification confidence ≥80%, SLR ≥8.

### Level 2 — Trained Matching (Campaigns 50–500)

- **Verification:** Same Day-1 stack, thresholds tuned on ConversionOracle v1 data.
- **Disclosure:** DisclosureBot v1 (AI-assisted pre-screen).
- **Matching:** Agent trained on **verified performance data** (creator × category × offer × geo × time → conversion). Human review drops to spot-checks below confidence <0.70.
- **Campaign design:** Agent generates brief from templates derived from top-performing Williamsburg Coffee+ campaigns. Human edits only on custom requests.
- **Ops cost per campaign:** ~1–2 human hours.
- **Software Leverage Ratio:** 12 (M6 target)

**Exit signal:** Matching prediction vs. actual conversion within ±15%, merchant-approved-slate rate ≥80%, ConversionOracle v1 A/B lift ≥15%, SLR ≥12.

### Level 3 — Autonomous Operation (Campaigns 500+)

- **Verification:** Same Day-1 stack; thresholds tighter as fraud patterns learned.
- **Disclosure:** DisclosureBot v2 (adaptive per channel convention).
- **Matching:** Agent auto-dispatches on standard requests. Human review only for novel vertical, novel neighborhood, or high-value (>$10K GMV) campaigns.
- **Campaign design → execution → payout → reporting:** End-to-end agent-run for typical campaigns.
- **Dashboard:** Merchants see real-time state; comms move from email to in-app.
- **Ops cost per campaign:** <30 human minutes typical, 2–4 hours on edge cases.
- **Software Leverage Ratio:** ≥25 (M12 target) → ≥50 (M24 target)

**Exit signal:** ≥90% of campaigns end-to-end without human intervention; satisfaction/repeat rate holds or rises; ConversionOracle v2 LTV prediction error ≤30%.

### Escalation / Fallback (Always-On)

Humans always handle:
- Novel merchant vertical or neighborhood (no prior data)
- High-value disputes (>$10K impact)
- Systemic fraud (new attack vector, unmodeled)
- Creator grievances / appeals (human is court of record)
- FTC / legal inquiries (48h output full audit trail; 72h external counsel engaged; 90-day public transparency report if incident material)

## Risk Mitigation

### Verification Risk — Gamed AI / Synthetic Proofs
Human spot-check 10% of auto-approves continuously. Fraud playbook updated monthly. Multi-signal verification (vision + OCR + geo + velocity) — gaming all four leaves tells. False-positive <2%, false-negative <5%.

### Matching Risk — Bad Slate → Merchant Blames Push
Merchant sees slate **before** dispatch and can edit. Confidence bands shown on every match. Early-stage campaigns get human second-look on every slate. ConversionOracle A/B Month-6 commitment ensures lift claims are evidenced.

### Autonomy Risk — Scaling Before Quality Holds
Each level-up (1→2, 2→3) has hard exit signals. If satisfaction or verification accuracy drops, we revert a level and retrain. We do not push through.

### Narrative Risk — "AI-Washing" Accusations
Day-1 AI is **demonstrable** — a reporter or investor can watch a live verification + matching + DisclosureBot run in <5 minutes. We never claim autonomy beyond what we can show. And if Month-6 ConversionOracle A/B lift <15%, we honestly pivot the moat narrative to ops leverage + supply density + architectural FTC compliance.

### Compliance Risk — FTC Action
DisclosureBot = architectural compliance. $1M E&O + $25K Y1 reserve + quarterly external audit + 48h audit-trail export on inquiry. This is diligence-grade, not aspirational.

### Bus Factor — Z as Sole Engineering
ML Advisor Week 6–8. Senior ML eng Month 9. Code doc + README per module SOP from Day 1. Founder equity standard 4y vest + 1y cliff + reverse vesting protects the company.
